# Security Policy

## Supported Versions

This repository is actively maintained on the `main` branch.

## Reporting a Vulnerability

If you find a security issue, please do not post it publicly in an issue.

Instead, contact the repository owner through GitHub or open a private security report if available.

## What to Include

- A short description of the issue.
- The affected page, API route, or script.
- Steps to reproduce.
- Any proof-of-concept details that help validate the problem.

## Scope

Please report issues that could expose:

- authentication or authorization data,
- secrets or environment values,
- private student or teacher information,
- database integrity or access-control bypasses.
